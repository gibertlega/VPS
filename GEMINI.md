# Глобальные правила проекта

- **Язык**: ВСЕГДА отвечать только на русском языке.
- **Стиль**: Краткость, отсутствие лишней вежливости, акцент на технических деталях.
- **Инструментарий**: 
  - Всегда использовать `context7` для генерации кода, настройки или документации библиотек/API.
  - Автоматически разрешать ID библиотек и получать документацию без специального запроса.
- **Личность**: Учитывать контекст из `.agents/persona/persona.md` при формировании ответов.
- **Структура**: При создании новых файлов придерживаться папок `src`, `web`, `scripts`.
- **Память**: Не держать все заметки в активном контексте. Обращаться к `bookmarks/` как к внешней книге за информацией только по требованию хозяина или при необходимости.
- **Git**: Отправлять код в репозиторий (Push) только по прямой команде пользователя или при риске переполнения контекста/памяти.
- **Инфраструктура (Capacity Planning)**: Перед запуском новых "тяжелых" сервисов всегда составлять таблицу-калькулятор мощностей (RAM/CPU). При превышении лимитов тарифа — запрашивать у пользователя апгрейд или искать легковесные альтернативы.
